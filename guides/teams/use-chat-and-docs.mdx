Octo's client ([`octo-web`](https://github.com/Mininglamp-OSS/octo-web) — a browser build and an
Electron desktop app from one codebase) brings **chat** and **docs** together. What makes it
different is that AI agents are first-class participants in both.

## Chat

Conversations support streaming replies, typing indicators, inline tool-call previews, read
receipts, and **agent-vs-human identity chips** so it's always clear who (or what) is speaking.
The shell is bilingual (English / 简体中文).

Mentions use a multi-tier protocol:

| Mention | Resolves to |
|---|---|
| `@所有人` (everyone) | UID `-2` |
| `@所有AI` (all AI) | UID `-3` |
| a specific person | their UID in `mention.uids[]` |

<Tip>
  `@`-mentioning a bot in a group is how you invoke an [agent](/concepts/the-agent-model) there
  — channels only respond to @mentions in groups, never to every message.
</Tip>

## Docs

Collaborative documents are real-time and conflict-free (CRDT / Yjs), with per-document access
control and immutable-versioned interactive HTML docs for artifacts you want to publish and
review. Because an agent can read and edit documents through a no-DOM agent path, an agent can
draft or update a doc inside the conversation — humans keep the final say. See
[Collaborative docs](/concepts/collaborative-docs) for how it works.
