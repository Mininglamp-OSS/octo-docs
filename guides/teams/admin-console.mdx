**[`octo-admin`](https://github.com/Mininglamp-OSS/octo-admin)** is the operator console for an
Octo deployment — tenants, orgs, users, channels, and Lobster agent configuration in one pane.
It's a thin React front-end over `octo-server`'s `/admin/*` REST APIs, with no business logic or
database of its own.

## Access

With the [Compose deployment](/guides/operators/deploy-compose), the console is served at:

```bash
http://<your-host>:28080/admin/     # log in as superAdmin
```

Operator authentication is delegated to `octo-server` (SSO or an admin token), and **every
mutation passes backend-side RBAC** — the UI is a convenience, not the security boundary. Deploy
it as a separate, internal/VPN-only, SSO-gated bundle.

## Self-hosting the console

It's a Vite app:

```bash
pnpm dev        # dev server; point VITE_ADMIN_API_* at your octo-server
pnpm build      # static bundle → ./dist
```

Retarget the API via `.env.local` (`VITE_ADMIN_API_*`). The console ships EN / 中文 locales.

<Info>
  Because the admin console holds no data and enforces nothing on its own, the security posture
  is entirely: **serve it privately, gate it behind SSO, and rely on server-side RBAC.** See
  [Security hardening](/guides/operators/security-hardening).
</Info>
