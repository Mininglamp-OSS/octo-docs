Beyond the core stack, Octo has optional subsystems you turn on as needed. Search and summary are
**deployment profiles**; mail and speech are **standalone services**.

## Message search

Search (Kafka + OpenSearch + `es-indexer`) is **off by default** across Compose, Kustomize, and
Helm. Enabling it is two phases — provision, then cut over:

<Steps>
  <Step title="Provision the infrastructure">
    ```bash
    ./setup.sh --search        # writes COMPOSE_PROFILES=search (merges with --summary)
    sudo ./setup.sh --up
    ```
    On Helm set `search.enabled: true`; on Kustomize apply `kustomize/search`.
  </Step>
  <Step title="Index history and flip the reader (zero-downtime)">
    ```bash
    cd docker && scripts/search-upgrade.sh
    ```
    The script runs a gated cutover (seed the cursor to the high-watermark → producer on →
    backfill + reconcile → bind the alias → switch the reader onto OpenSearch).
  </Step>
</Steps>

See [Search & export pipeline](/concepts/search-and-export-pipeline) for the architecture and
[Export and search messages](/guides/integrators/export-and-search-messages) for the caller API.

## LLM summary

The smart-summary services are off by default:

```bash
./setup.sh --summary       # writes COMPOSE_PROFILES=summary; requires LLM_API_KEY
```

On Helm, `summary.enabled: true`. The provider is configured via `LLM_API_KEY` (provider-agnostic
in Compose). When enabled, `/summary/health` becomes a live probe.

## Mail

**[`octo-mail`](https://github.com/Mininglamp-OSS/octo-mail)** is a multi-tenant, horizontally
scalable mail-server kernel (IMAP4rev2 · JMAP · SMTP · REST) built around an append-only,
per-account change-log on PostgreSQL + S3. It runs as its own service; Helm exposes SMTP support
configuration. Deploy and scale it independently of the IM stack.

## Speech-to-text

**[`octo-speech`](https://github.com/Mininglamp-OSS/octo-speech)** is a standalone speech-to-text
microservice.

<Note>
  Speech is **not** a Compose/Helm profile in `octo-deployment` — deploy `octo-speech` as its own
  service and point `octo-server` at it. (The documented deployment profiles are summary and
  search; mail and speech are separate services.)
</Note>
