**[`octo-marketplace`](https://github.com/Mininglamp-OSS/octo-marketplace)** is the control plane
for the Octo **Skill and MCP marketplace** — a place to publish, version, and distribute
[Skills](/guides/bot-developers/publish-a-skill) and MCP servers.

<Warning>
  **Maturing service.** The Skill lifecycle (upload / parse / create / version / download,
  categories, tags) and the MCP catalog (CRUD, probe, icons, categories) are implemented in the
  current spec and code. Catalog discovery UI, publishing/versioning polish, and full database
  persistence are still being wired — treat production use as early.
</Warning>

## Where it sits

```mermaid
graph LR
  web[octo-web] --> mkt[marketplace-api]
  cli[octo-cli] --> mkt
  server[octo-server] <--> mkt
```

Ownership is split cleanly:

- **`octo-server`** owns identity (it authenticates users and `bf_` bots).
- **`octo-marketplace`** owns assets, releases, and policy.
- **`octo-cli`** owns local install on the consumer side.

## Run the service

```bash
go run ./cmd/marketplace-api      # port 8092
# or
docker compose up --build         # MySQL on 3306
```

It uses Go 1.25 + Gin + MySQL, and follows the API-service shape of
[`octo-smart-summary`](/ecosystem/repository-guide). Auth is unified user + `bf_` Assistant,
fails closed (`AUTH_ENABLED` defaults true), via the Octo auth client. When mounted behind the
web gateway it serves under `/market/api/v1`.
