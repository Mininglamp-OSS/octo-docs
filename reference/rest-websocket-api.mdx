The Octo bot-facing REST API spans **84 operations across 8 domains**. Each domain in the sidebar is an **interactive reference** generated from the OpenAPI 3.1 spec embedded in [`octo-cli`](https://github.com/Mininglamp-OSS/octo-cli) — the same spec that drives the CLI and the server, so it never drifts from the implementation. Open any endpoint to inspect its schema and send a live request.

<Tip>
  Call these endpoints with a bot token (`bf_…`) as a bearer credential — set your instance URL and token in an endpoint's **Try it** panel to send a live request, or use [`octo-cli`](/guides/bot-developers/drive-octo-with-cli). See [Verify credentials with octo-auth](/guides/integrators/verify-credentials-with-octo-auth). The entities and enums these operations read and write are catalogued in the [data model](/reference/data-model).
</Tip>

## Risk levels

Each operation is tagged with an `x-octo-risk` level from the spec:

- 🟢 **read** — no state change.
- 🟠 **write** — creates or mutates state.
- 🔴 **high-risk** — destructive or broadly-scoped; guard carefully.
