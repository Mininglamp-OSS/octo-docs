The Octo bot-facing REST API spans **84 operations across 8 domains**. Every domain page is generated directly from the OpenAPI 3.1 spec embedded in [`octo-cli`](https://github.com/Mininglamp-OSS/octo-cli) — the same spec that drives the CLI and the server, so this reference never drifts from the implementation.

<Tip>
  The easiest way to call these endpoints is via [`octo-cli`](/guides/bot-developers/drive-octo-with-cli) (each operation maps to a `CLI op` shown on its domain page) or directly over HTTP with a bot token (`bf_…`). See [Verify credentials with octo-auth](/guides/integrators/verify-credentials-with-octo-auth).
</Tip>

## Risk levels

Each operation is tagged with an `x-octo-risk` level from the spec:

- 🟢 **read** — no state change.
- 🟠 **write** — creates or mutates state.
- 🔴 **high-risk** — destructive or broadly-scoped; guard carefully.
