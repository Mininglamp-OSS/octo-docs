Octo 面向机器人的 REST API 覆盖 **8 个领域、共 84 个接口**。每个领域页面都直接由内嵌于 [`octo-cli`](https://github.com/Mininglamp-OSS/octo-cli) 的 OpenAPI 3.1 规范生成 —— 与驱动 CLI 和服务端的是同一份规范,因此本参考永不与实现漂移。

<Tip>
  调用这些接口最简单的方式是使用 [`octo-cli`](/zh/guides/bot-developers/drive-octo-with-cli)(每个接口都对应领域页中标注的 `CLI 命令`),或用助理 token(`bf_…`)直接走 HTTP。参见 [用 octo-auth 校验凭证](/zh/guides/integrators/verify-credentials-with-octo-auth)。这些操作读写的实体与枚举,汇总于[数据模型](/zh/reference/data-model)。
</Tip>

## 风险级别

每个接口都带有来自规范的 `x-octo-risk` 级别标记:

- 🟢 **读** —— 不改变状态。
- 🟠 **写** —— 创建或修改状态。
- 🔴 **高危** —— 破坏性或作用域较广,请谨慎。
