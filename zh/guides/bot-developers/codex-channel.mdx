**[`codex-channel-octo`](https://github.com/Mininglamp-OSS/codex-channel-octo)** 是一个
独立的 Node.js 网关，它把 [OpenAI Codex](https://developers.openai.com/codex)
（通过 `@openai/codex-sdk`，其中已捆绑运行时）桥接进 Octo IM。它镜像其同胞
[`cc-channel-octo`](/zh/guides/bot-developers/claude-code-channel)；只是智能体内核不同。

<Warning>
  **早期测试版本（v0.1.0）。** 核心路径——注册、连接、多轮聊天、
  多机器人——已经过验证，但**尚无 cron，也尚无技能**。若需一个功能完整的
  频道，请使用 [cc-channel-octo](/zh/guides/bot-developers/claude-code-channel)。
</Warning>

## 运行它

<Warning>
  npm 包 `@mininglamp-oss/codex-channel-octo` **尚未发布**到公共 registry(这是早期 v0.1.0 测试版）。
  在发布之前请从源码安装：
</Warning>

```bash
git clone https://github.com/Mininglamp-OSS/codex-channel-octo.git
cd codex-channel-octo
npm install
npm run build       # tsc → dist/
npm start           # 或：node dist/（参见仓库 README）
```

发布后,一行命令即可：`npm install -g @mininglamp-oss/codex-channel-octo` → `codex-channel-octo start`。

需要 Node ≥ 22 以及一个 Octo 机器人令牌（`bf_…`）。该网关注册为一个 Octo 机器人、
通过 WebSocket 连接、用本地 Codex 运行一项编码任务，然后把回复回发。它
支持私信（DM）、群组（@提及）和社区话题、多轮对话、在一个进程中运行多个机器人，
以及配置热重载。

## Codex 线程互通

每个 Octo 会话都映射到一个 **Codex 线程（thread）**，因此一个会话可以从你的
终端恢复：

```bash
codex exec resume <threadId>
```

这种 IM ↔ 终端 的互通是可选启用的——你可以在本地 Codex
会话中接续一段聊天对话，反之亦然。

## 流式行为

与 cc-channel-octo 的逐块流式不同，Codex 会**整体投递最终消息**，
中间步骤则作为进度更新呈现（而非逐 token）。

## 安全

- `sandboxMode` 默认为 **`read-only`**；启用写入需要经过一次刻意的双重切换。
- `danger-full-access` 会被直接拒绝。
- 每个机器人拥有自己的 `CODEX_HOME`，从而按机器人隔离 Codex 状态。
