在核心技术栈之外，Octo 还有一些可按需开启的可选子系统。搜索和摘要是
**部署配置档（profiles）**；邮件和语音是**独立服务**。

## 消息搜索

搜索（Kafka + OpenSearch + `es-indexer`）在 Compose、Kustomize 和
Helm 中**默认关闭**。启用它分两个阶段——先供给，再切换：

<Steps>
  <Step title="供给基础设施">
    ```bash
    ./setup.sh --search        # writes COMPOSE_PROFILES=search (merges with --summary)
    sudo ./setup.sh --up
    ```
    在 Helm 上设置 `search.enabled: true`；在 Kustomize 上应用 `kustomize/search`。
  </Step>
  <Step title="索引历史并切换读取端（零停机）">
    ```bash
    cd docker && scripts/search-upgrade.sh
    ```
    该脚本执行一次受控切换（将游标播种到高水位线 → 开启生产者 →
    回填 + 对账 → 绑定别名 → 将读取端切换到 OpenSearch）。
  </Step>
</Steps>

参见 [搜索与导出流水线](/zh/concepts/search-and-export-pipeline) 了解架构，参见
[导出与搜索消息](/zh/guides/integrators/export-and-search-messages) 了解调用方 API。

## LLM 摘要

smart-summary 服务默认关闭：

```bash
./setup.sh --summary       # writes COMPOSE_PROFILES=summary; requires LLM_API_KEY
```

在 Helm 上，`summary.enabled: true`。提供商通过 `LLM_API_KEY` 配置（在 Compose 中
与提供商无关）。启用后，`/summary/health` 会变为活跃探测端点。

## 邮件

**[`octo-mail`](https://github.com/Mininglamp-OSS/octo-mail)** 是一个多租户、可水平
扩展的邮件服务器内核（IMAP4rev2 · JMAP · SMTP · REST），围绕基于 PostgreSQL + S3 的
仅追加、按账户变更日志构建。它作为独立服务运行；Helm 暴露了 SMTP 支持
配置。请独立于 IM 技术栈来部署和扩容它。

## 语音转文字

**[`octo-speech`](https://github.com/Mininglamp-OSS/octo-speech)** 是一个独立的语音转文字
微服务。

<Note>
  语音在 `octo-deployment` 中**不是** Compose/Helm 配置档——请将 `octo-speech` 作为独立
  服务部署，并让 `octo-server` 指向它。（已文档化的部署配置档是 summary 和
  search；邮件和语音是单独的服务。）
</Note>
